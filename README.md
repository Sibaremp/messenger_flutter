# Caspian College Messenger

Мессенджер для студентов и преподавателей колледжа, разработанный на Flutter.
Адаптивный интерфейс для мобильных устройств и десктопа.

---

## Возможности

### Чаты
- **Личные сообщения** — переписка один на один
- **Группы** — чаты, в которых могут писать все участники
- **Сообщества** — каналы с правом записи только у администратора, комментарии к постам
- **Разделы**: «Общение» (студенты) и «Академический» (преподаватели)
- Ответ на сообщение (swipe-to-reply с анимацией и вибрацией)
- Пересылка, редактирование и удаление сообщений
- Режим выделения нескольких сообщений
- Вложения: фото, видео, документы
- Полноэкранный просмотр медиа с видеоплеером
- **@упоминания** участников с подсветкой и уведомлениями
- **Опросы** — создание, голосование, завершение

### Комментарии (сообщества)
- Полная функциональность: ответ, редактирование, удаление, пересылка
- Вложения в комментариях
- Режим выделения комментариев

### Статус сообщений
| Иконка | Статус |
|--------|--------|
| ⏱ | Отправляется |
| ✓ | Отправлено |
| ✓✓ | Доставлено |
| ✓✓ (голубые) | Прочитано |
| ✗ | Ошибка |

Статусы отображаются в пузырях сообщений и в списке чатов.

### Аватары и имена
- 8 контрастных hash-based цветов для имён отправителей
- Группы и сообщества: **инициалы из названия** вместо иконки (как в Telegram)
- Автоматический fallback на инициалы при ошибке загрузки фото
- Учебная группа студента отображается перед именем в групповых чатах
  (например: **ИС-22 Алексей**)

### Непрочитанные сообщения
- Бейдж с числом непрочитанных на каждом чате в списке
- После открытия чата счётчик сбрасывается (`POST /chats/{id}/read`)

### Уведомления
- Вкладка уведомлений хранит упоминания (`@ник`, `@all`) и ответы на сообщения
- Подсчёт непрочитанных уведомлений на иконке вкладки
- Каждое уведомление помечается типом: «Упоминание» / «Ответ»

### Профили групп и сообществ
- Шапка с фото (или градиентный плейсхолдер с инициалами при отсутствии фото)
- Список участников с ролями: Создатель / Администратор / Участник
- Текущий пользователь всегда отображается как «Вы»

### Контакты и профили
- Автоматическая загрузка телефонной книги устройства (Android / iOS)
- Маркировка контактов, зарегистрированных в приложении
- Профиль собеседника: имя, описание, телефон, учебная группа
- Профили групп и сообществ: участники, роли, описание

### Профиль пользователя
- Аватарка (камера или галерея)
- Имя, логин, «О себе», учебная группа
- Телефон и группа — read-only после регистрации
- Автозаполнение номера с SIM-карты (Android)
- Переключатель светлой / тёмной / системной темы

### Управление устройствами (сессии)
- Список активных сессий с платформой и временем последней активности
- Текущая сессия помечается «Это устройство»
- Завершение конкретной сессии или всех сразу

### Авторизация
- Вход и регистрация с выбором роли (студент / преподаватель)
- Выбор учебной группы при регистрации (только для студентов)
- Безопасное хранение сессии (`flutter_secure_storage`)
- Переключение темы на экране входа

### Адаптивный интерфейс
- **Мобильные устройства** — полноэкранная навигация с BottomNavigationBar
- **Десктоп** (ширина ≥ 800 px) — трёхпанельная раскладка:
  боковая навигация, список чатов, область переписки + панель профиля
- Устройства (сессии) открываются **встроенно** в панели профиля (не полноэкранно)

### Слой сервисов

`ChatService` — абстрактный интерфейс с потоком событий `Stream<ChatEvent>`.

| Реализация | Описание |
|---|---|
| `LocalChatService` | Данные в памяти (для разработки без сервера) |
| `ApiChatService` | REST API + SignalR (для работы с сервером) |

Переключение между режимами — в `main.dart`.

## Запуск

```bash
flutter pub get
flutter run
```

### Android — разрешения
Прописаны в `AndroidManifest.xml`:
- `INTERNET` — сетевые запросы
- `READ_CONTACTS` — телефонная книга
- `READ_PHONE_STATE` / `READ_PHONE_NUMBERS` — данные SIM-карты
- `READ_MEDIA_IMAGES` / `READ_MEDIA_VIDEO` — медиа (Android 13+)
- `CAMERA` — съёмка аватара

## Подключение к серверу

1. Разверните серверную часть (C# .NET / ASP.NET Core с SignalR)
2. В `lib/services/api_config.dart` укажите адрес:
   ```dart
   // Для постоянного IP (реальное устройство / продакшн):
   ApiConfig.overrideHost = '192.168.1.100';
   ```
   Без переопределения:
   - Android-эмулятор → `10.0.2.2` (маршрут к `localhost` хоста)
   - Desktop / iOS → `localhost`
3. В `lib/main.dart` убедитесь, что используется `ApiChatService`
4. Запустите приложение

> **Важно:** Если сервер возвращает абсолютные URL с `localhost` (например, в путях медиафайлов),
> `ApiConfig.resolveMediaUrl` автоматически заменяет хост на сконфигурированный —
> аватарки и вложения корректно загружаются на реальных устройствах.
